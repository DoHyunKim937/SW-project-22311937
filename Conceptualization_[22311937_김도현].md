1. Business Purpose
이 소프트웨어의 목적은 최근 주식시장의 인기가 급격히 올라감에따라 투자자들이 기업의 재무 지표를 알아볼 수 있게 지원하는 것이 목표이다.
지금의 금융 서비스는 복잡한 데이터들로 이루어져있기 떄문에 투자자들이 이를 쉽게 이해하기가 힘들다.
따라서 이 프로젝트는 기업의 핵심 지표인 per,roe,psr 등을 단순화하여 표현하고 직관적인 해석을 추가하여 사용자를 도울 수 있게 하는 것이다.

2.System context diagram
 기업 선택(select), per 조회(search), per 해석 조회(analysis), 기업 정보 표시(impormation), 기업 비교(comparison) , 데이터 초기화(reset) , 추가 지표 조회 (plus)

3.Use case list
 기업 선택(select) => 사용자가 기업 목록에서 하나를 선택한다.
 per 조회(search) => 선택한 기업의 per 값을 확인한다.
 per 해석 조회(analysis) => per 값에 관한 직관적인(고평가,적절,저평가 등) 평가를 제공한다.
 기업 정보 표시(impormation) => 선택한 기업의 정보(하는 일 등)을 화면에 표시한다.
 기업 비교(comparison) => 두개 이상의 기업을 선택하여 per 값을 비교한다.
  데이터 초기화(reset) => 선택한 기업 및 표시된 데이터 값들을 정리하고 리셋한다.
 추가 지표 조회 (plus) => per 외에 여러 지표 roe , pbr , psr 와 같은 재무 지표를 조회한다.

4. Concept of operation
기업 선택(select)
- purpose => 사용자가 조회하고자하는 기업을 선택할 수 있도록한다.             
-approach => 사용자는 어플리케이션 실행 후 제공되는 기업 목록에서 원하는 기업을 선택한다. 시스템은 해당 선택을 인식하고 이후 데이터 조회를 준비한다.
-dynamics => 앱 실행 후 기업을 선택하는 경우
-goals    => 기업 선택 기능을 구현한다.            
                
 per 조회(search) 
 - purpose =>  선택한 기업의 per 값을 사용자에게 제공한다.            
-approach => 사용자가 기업을 선택하면 시스템은 해당 기업의 데이터를 조회하고 per 값을 화면에 출력한다.
-dynamics => 기업 선택 후 per 조회 요청하는 경우
-goals    => per 조회 기능을 구현한다.
    
 per 해석 조회(analysis)
- purpose => per 값의 의미를 초보 투자자도 이해할 수 있도록한다. 그리고 그 기업의 해석을 도와준다     
-approach => 시스템은 per 값에 따라 미리 정의된 기준을 적용하여 저평가,적절,고평가 등의 해석 결과를 사용자에게 제공한다.
-dynamics => per 값 표시 이후 해석 제공하는 경우
-goals    =>  per 해석 기능을 구현한다.

 기업 정보 표시(impormation)
- purpose =>  사용자가 선택한 기업의 기본 정보를 확인할 수 있도록 한다.            
-approach => 시스템은 기업 데이터에서 이름 등의 기본 정보를 불러와 화면에 함께 표시한다.
-dynamics => 기업 선택을 하는 경우 정보 표시
-goals    => 기업 정보 표시 기능을 구현한다.

 기업 비교(comparison)
- purpose => 여러 기업의 per 값을 비교하여 상대적인 가치를 파악할 수 있도록 한다.              
-approach => 사용자가 두 개 이상의 기업을 선택하면 시스템은 각 기업의 per 값을 조회하여 비교 형태로 출력한다.
-dynamics => 복수 기업 선택하는 경우 비교 수행
-goals    =>  기업 비교 기능을 구현한다.
  
  데이터 초기화(reset)
  - purpose =>  사용자가 다시 기능을 사용하기 위해 현재 선택 및 표시된 데이터를 초기 상태로 되돌릴 수 있도록 한다.            
-approach => 사용자가 리셋 버튼을 누르면 시스템은 모든 선택 상태와 출력 데이터를 제거하고 초기 화면으로 복원한다.
-dynamics => 초기화 버튼을 클릭한 경우
-goals    => 데이터 초기화 기능을 구현한다.

 추가 지표 조회 (plus)
 - purpose =>  per 외 추가적인 재무 정보( roe , pbr , psr)을 제공하여 이해도를 높인다.            
-approach => 사용자가 추가 지표 조회를 요청하면 시스템은 roe pbr psr 계산에 필요한 데이터들을 불러와 화면에 출력한다.
-dynamics => 추가 지표 조회를 요청할 경우
-goals    =>  추가 지표 조회 기능을 구현한다.
